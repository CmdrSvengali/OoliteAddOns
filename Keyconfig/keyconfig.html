Help▲Top

The virtual keyboard uses a US keyboard layout with the addition of the key between left shift and z (a evil twin of the last key in the qwerty row). Additional layouts for French, German, Italian and Russion keyboards are included. Generated output uses the US layout, so don't get confused.

The special keys Escape, Tab, Cursor keys (e.g. CR^ ), Function keys, Home, End, Delete, Insert, Page up, Page down or the Numpad are only available through the virtual keyboard. For Mac-user some special keys are defined as symbols. The left mouse button (MB LeftClick and MB DoubleClick) can be mapped as well.

The checkbox for Assigned keys toggles the coloring and override protection for already used or hardcoded keys in Oolite (e.g. Q). Double-assignment of keys is allowed as long as their functionality doesn't collide, but if you really want to do it switch it off. Additional possibilitites are available by activating the Legend/Options.

When done click the 'Create configuration' button and copy the content into your keyconfig.plist.
Note: Do not use Notepad under Windows - it may corrupt your file. Wordpad seems to work fine, but there are
lots of other editors available (e.g. Notepad++). Oolite expects the file without BOM.

The easiest way is probably to create a folder named User.oxp in your AddOns directory, containing one folder named Config. Create a file named keyconfig.plist there or copy Oolites keyconfig.plist into it, then replace the content with the generated output.
